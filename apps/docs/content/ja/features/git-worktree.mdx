# Git Worktree 統合

Tsunagi は [git worktree](https://git-scm.com/docs/git-worktree) を自動で作成・管理します。各タスクは独立したチェックアウトに住み、同じオブジェクトデータベースを共有します。

## レイアウト

```
~/.tsunagi/workspaces/<owner>/<repo>/
├── .bare/                    # bare clone
├── .default/                 # デフォルトブランチのリファレンスチェックアウト
└── <branch-dir>/             # タスクブランチごとに 1 つのディレクトリ
```

## Tsunagi が自動化すること

- リポジトリ登録時の **`git clone --bare`**
- タスク作成時の **`git worktree add`** (適切なベースブランチで)
- タスク削除時の **`git worktree remove`**
- `git worktree add --orphan` による **空リポジトリの扱い** (Git ≥ 2.42 が必要)
- `origin/HEAD` からの **デフォルトブランチ検出** (`main` / `master` への fallback あり)
- リモートに最初の commit が来た時点での、orphan デフォルト worktree の **detached HEAD への自動 upgrade**

## なぜタスクごとの worktree なのか

複数の Claude セッションが完全に並列で動作しても、ファイルシステム上で衝突しません。それぞれが独自の作業ディレクトリ・独自の `.git/index`・独自のブランチを持ちます。
