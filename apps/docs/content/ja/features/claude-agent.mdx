# Claude エージェント & タブセッション

Tsunagi の各タスクは **複数の Claude Code セッション** をホストできます。これを **タブ** と呼びます。タブはタスクの worktree 内で Claude を動かす PTY バックエンドのターミナルセッションです。

## なぜ複数タブか

- 機能を実装するタブとテストをデバッグするタブを並行して動かす
- メインの会話を失うことなく、使い捨ての探索的セッションを起動する
- 2 つのアプローチを横並びに比較する

すべてのタブが同じ worktree を共有するので、1 つのタブからのファイル編集は他のタブにも即座に見えます。

## タブの起動方法

**+ New Tab** をクリックすると、Tsunagi は以下を行います:

1. タスクの worktree 内で `node-pty` 経由でシェルプロセスを起動
2. Tsunagi 設定画面で定義した環境変数をセット
3. そのシェルで Claude Code を起動 (Claude Code 自身が使うのと同じ認証情報で)
4. Claude の hooks を Tsunagi バックエンドに接続して UI から状況を見えるように

## 耐久性

タブはブラウザのリロードや短時間のネットワーク切断を跨いで永続化されます。ブラウザを閉じてもタブはサーバー側で動き続け、Tsunagi を再度開けば再接続できます。

<Callout type="info">
**準備中**: タブの状態、内部 PTY マネージャ、クラッシュしたセッションの復旧セマンティクスのリファレンス。

Claude Code 自体については [Claude Code 公式ドキュメント](https://docs.claude.com/en/docs/claude-code/overview) を参照してください。
