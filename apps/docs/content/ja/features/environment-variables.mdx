# 環境変数管理

多くのプロジェクトは追加の環境変数 (データベース URL、API トークン、ローカル feature flag) を必要とします。Tsunagi では 3 つのスコープで環境変数を定義でき、各 Claude セッションに自動でマージされます。

## スコープ

| スコープ       | 適用対象                                     | 用途                                       |
| -------------- | -------------------------------------------- | ------------------------------------------ |
| **グローバル** | すべての Claude セッション                   | 全体で必要なもの (`EDITOR` 等)             |
| **オーナー**   | GitHub organization 配下のすべてのリポジトリ | org 全体のシークレット (`AWS_PROFILE` 等)  |
| **リポジトリ** | 特定の 1 つのリポジトリ                      | プロジェクト固有の設定 (`DATABASE_URL` 等) |

## マージ順

変数はスコープが具体的になる順にマージされます:

```
グローバル → オーナー → リポジトリ → (リクエスト時の上書き)
```

後のソースが先のソースを上書きするので、リポジトリスコープの値はグローバルの同名値より優先されます。

## どこに保存されるか

Tsunagi のローカル SQLite データベースに `(scope, owner, repo, key)` でスコープして保存されます。設定ページから管理できます (将来的には REST API からも管理可能になります)。

<Callout type="warning">
  環境変数はプレーンな SQLite に保存されます。`~/.tsunagi/state/tsunagi.db`
  を他のシークレットを含むファイルと同じように扱ってください——慎重にバックアップし、git
  にチェックインしないこと。
</Callout>
