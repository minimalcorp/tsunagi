# 環境変数管理

Tsunagi が起動する Claude セッションはすべてシェルの中で動きます。多くのプロジェクトはそのシェルに特定の環境変数が必要です——データベース URL、API トークン、feature flag、プロファイル名など。Tsunagi ではこれらの変数を 1 箇所で定義し、適切にスコープされた形で各セッションに自動注入できます。

## 3 つのスコープ

変数は 3 つのスコープで定義できます。広い順から:

| スコープ       | 適用対象                                            | 典型的な用途                                                                         |
| -------------- | --------------------------------------------------- | ------------------------------------------------------------------------------------ |
| **グローバル** | リポジトリを問わずすべての Claude セッション        | デフォルトの `EDITOR` や `PAGER` 等、すべてで使いたいもの                            |
| **オーナー**   | 1 つの GitHub organization 配下のすべてのリポジトリ | org 全体で共通のシークレット (例: `my-org` 配下の全 repo で共有される `AWS_PROFILE`) |
| **リポジトリ** | 特定の 1 つの `owner/repo` ペア                     | プロジェクト固有の設定 (`DATABASE_URL`、`STRIPE_SECRET_KEY` 等)                      |

3 つのスコープすべてを **Settings → Environment Variables** の同じ画面から管理できます。

## マージ順

Tsunagi が Claude セッションを起動するとき、スコープの具体性が高くなる順に変数をマージして、その結果をシェルに渡します:

```
グローバル → オーナー (タスクの repo のもの) → リポジトリ (タスクの repo) → シェル
```

後のソースが先のソースを **上書きします**。リポジトリスコープの `DATABASE_URL` は同じキーのグローバル `DATABASE_URL` に勝ちます。これにより「デフォルトはグローバルに置き、必要に応じて org や repo で上書きする」というシンプルなメンタルモデルが使えます。

ログインシェル環境 (`~/.zshrc` 等で設定されているもの) にも既にある変数はそのまま継承されます——Tsunagi は周囲のシェル環境を置き換えるのではなく追加します。

## ターミナルへの自動注入

これらの変数を使うのに特別な操作は不要です。タスクで新しいタブを開くたびに、Tsunagi はそのタスクの `(owner, repo)` に対するマージ結果を計算し、Claude Code を起動する前に PTY の環境にセットします。Claude から見ると、あなたがシェルの rc ファイルで export しているのと全く同じ見え方です。

既に開いているタブがある状態で変数を編集した場合、変更は **新しいタブにのみ** 適用されます。既存タブは起動時の環境を保持するので、変更を反映するにはタブを一度閉じて開き直してください。

## Tsunagi が特別扱いする環境変数

いくつかの環境変数は Tsunagi または Claude Code 自身に特別な意味を持ちます。通常これらは Tsunagi の設定画面で定義するのではなく、Tsunagi を起動するログインシェルで設定するものですが、存在を知っておくと便利です。

| 変数                      | 用途                                                                                                                                                                                                                                    |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ANTHROPIC_API_KEY`       | Anthropic API の認証情報。Claude Code 自身が読む                                                                                                                                                                                        |
| `CLAUDE_CODE_OAUTH_TOKEN` | `ANTHROPIC_API_KEY` の代替となる OAuth トークン。Claude Code が読む                                                                                                                                                                     |
| `EDITOR`                  | Tsunagi が各 Claude タブの中でバンドル済みの Monaco ヘルパーに事前設定します。これにより `git commit` 等が `vi` の代わりに Monaco モーダルを開きます。グローバルスコープで Tsunagi 設定画面に `EDITOR` を定義するとこれを上書きできます |
| `TSUNAGI_SESSION_ID`      | 各 Claude タブのシェルに自動注入されます。スクリプトやプラグインがセッション内から現在のタスクを特定するのに使います                                                                                                                    |

`ANTHROPIC_API_KEY` や `CLAUDE_CODE_OAUTH_TOKEN` を Tsunagi の設定画面で定義するべきではありません——Tsunagi を起動するシェルで export しておけば、Claude Code が直接拾います (ターミナルから `claude` を手動起動する時と同じ)。

## データの保存先

Tsunagi UI で定義したすべての環境変数は、Tsunagi のローカル SQLite データベースに保存されます:

```
~/.tsunagi/state/tsunagi.db
```

各変数は `environment_variables` テーブルの行として `(scope, owner, repo, key)` でキー付けされています。

<Callout type="warning">
  このデータベースはあなたが入力したシークレットをそのまま保持します。 `~/.tsunagi/state/tsunagi.db`
  は機密ファイルとして扱ってください: 慎重にバックアップし、git
  にチェックインせず、マシンを共有する場合はファイルパーミッションを制限してください。
</Callout>

Tsunagi は値を暗号化して保存しません。暗号化が必要な場合は、シークレットをキーチェーン / vault に保存し、Tsunagi を起動するログインシェルから参照する形にして、Tsunagi のデータベースには保存しないでください。

## 個別変数の有効化・無効化

各変数には設定画面から切り替えられる **enabled** フラグがあります。変数を無効化すると、レコードは削除せずにマージ結果から除外されます。値自体を失わずに「一時的に OFF にする」(その変数なしで動作をテストしたい時等) のに便利です。
