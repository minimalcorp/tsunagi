# 4. レビューとステータス自動更新

Claude が hello world プログラムの実装を完了する頃には、タスクは以下のステータスを通過しているはずです:

```
backlog → coding → reviewing
```

あなたはタスクをドラッグもしていないし、ステータスボタンもクリックしていません。Tsunagi の plugin には **タスクステータスをいつ更新すべきか** を Claude に教える小さな skill が含まれていて、Claude が Tsunagi の MCP server を呼び出して更新するからです。

## ステータス遷移の意味

| ステータス  | 状態                                     |
| ----------- | ---------------------------------------- |
| `backlog`   | タスクは存在するがまだ誰も作業していない |
| `planning`  | アプローチについて人間と Claude が対話中 |
| `coding`    | Claude が積極的に実装中                  |
| `reviewing` | 実装完了、人間のレビュー待ち             |
| `done`      | タスク完了、人間が確認済み               |

hello world チュートリアルでは、タスクは **`reviewing`** にいるはずです。あなたを待っています。

## タスク一覧で確認

タスク一覧で自分のタスクを見つけてください。バッジが `reviewing` になっているはずです。プログレスバーは 100% です。

まだ `coding` のままなら、Claude がまだ作業中の可能性があります——少し待ってからもう一度確認してください。

何か問題が起きた場合は、タスクをクリックして詳細画面を開いてターミナル出力を読みましょう。よくある原因:

- 適切な言語がマシンに見つからなかった (まれ)
- Claude がプログラムを実行できなかった (sandboxing や権限の問題)

`please retry` または具体的な指示で Claude にリトライを依頼できます。
