# 2. Claude セッションを起動する

タスクが作成されたら、タスクをクリックして **タスク詳細画面** を開きます。ここが実際に Claude と作業する場所です。

## タブを作成する

タスク詳細で **+ New Tab** をクリックします。これで以下が起こります:

1. タスクの worktree 内で PTY バックエンドのシェルセッションを起動
2. Claude Code をそのシェルで起動 (Claude Code 自身が使うのと同じ認証情報で)
3. Claude の hooks を Tsunagi に接続して UI から状況を読めるように
4. タブにターミナルビューを開く

Claude がプロンプト待ち状態のシェルが見えるはずです。

## todo ベースのタスクで Claude にプロンプトを送る

以下のプロンプトを Claude セッションにペーストしてください:

```text
todo を作成して、このマシンに既にインストールされているプログラミング言語で "Hello, world!" を表示するプログラムを作成してください。出力が正しいことを確認するだけで OK で、commit や push は不要です。実装と確認が完了したら、タスクのステータスを `reviewing` に変更してください。
```

なぜ Claude に **最初に todo を作成させる** のでしょうか? Tsunagi は Claude の todo 更新を監視して、それを **タスクカード上のプログレスバー** として描画するからです。Claude が todo を作成した瞬間、UI にリアルタイムで現れるのが見えます。

## Claude が行うこと

通常 Claude は以下を行います:

1. **計画**: 「利用可能な言語を検出」「hello world プログラムを書く」「実行して出力を確認」のような todo リストを作成
2. **環境探索**: `which python3 / node / go / ruby / ...` を実行して実際にインストールされている言語を選ぶ
3. **実装**: `hello.py` のような小さなファイルを作成
4. **検証**: プログラムを実行して出力が `Hello, world!` であることを確認
5. todo を1つずつ **完了マーク**
6. Tsunagi の MCP server 経由で **タスクのステータスを `reviewing` に更新**

最後のステップが面白いところです——タスクが `coding` カラム (Claude が作業を開始した時点で自動的にここに来る) から `reviewing` カラムへ、あなたが何もしないのに移動するのが見えます。
