# 3. 進捗を監視する

メインのタスク一覧 (左上の **Tsunagi** ロゴ) に戻ります。タスクカードに、いくつかのライブ情報が表示されているはずです。

## 一目で見える情報

- **ステータス** — 色付きのバッジで `coding` (Claude が完了したら `reviewing`) を表示
- **プログレスバー** — Claude の todo リストから算出。Claude が各 todo を完了するたびにバーが伸びる
- **ブランチ** — このタスクの worktree が乗っているブランチ
- **Effort** — タスク作成時に設定したサイズ
- **Last updated time** — Claude の hook イベントからリアルタイム更新

## なぜこれが動くのか

Claude が `TodoWrite` (Claude 標準ツールの1つ) を呼び出すと、Claude Code の hook が発火します。hook は新しい todo を Tsunagi の Fastify バックエンドに POST し、バックエンドが Socket.IO 経由で接続中の Web UI にブロードキャストします。ブラウザがページリロードなしでプログレスバーを再描画します。

同じ仕組みで以下も処理されます:

- ステータス変更 (`backlog` → `coding` → `reviewing` → `done`)
- タブのアクティビティインジケーター
- 新しいツール呼び出し (ファイル書き込み、bash コマンド)

## 複数タスクを並列で動かせる

これが Tsunagi の本質的な目的です。2つ目のタブで2つ目のタスクを開いて別の Claude セッションにプロンプトを送ると——両方のプログレスバーが横に並んで動きます。1つのタスク一覧画面が「PM の操縦席」のままです。

このチュートリアルでは1つのタスクで十分です。
