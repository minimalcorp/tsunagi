# 5. 検証して完了

ここは人間のループです。作業を検証してタスクを `done` に動かします。

## worktree を確認する

ターミナルを開いて Claude が実際に何を作ったかを見てみましょう:

```bash
cd ~/.tsunagi/workspaces/<owner>/<repo>/feat-hello-world/
ls
```

`hello.py` (または `hello.js`、`hello.go`、Claude が選んだ言語次第) のようなものが見えるはずです。

実行してみます:

```bash
python3 hello.py
# Hello, world!
```

または Claude が使ったインタプリタで実行してください。出力はターミナルセッションで Claude が報告したものと一致するはずです。

## (任意) Tsunagi UI から確認する

Tsunagi でタスクのタブを開いて会話ログをスクロールすると、Claude が実際に何をしたかを正確に確認できます。ターミナルセッションは全コマンドと応答をキャプチャしています。

## タスクを done としてマークする

2 つの方法があります。

### Option A — Claude にお願いする

開いている Claude タブで以下のように入力します:

```text
プログラムが動作することを確認しました。タスクを done にしてください。
```

Claude は `tsunagi_update_task` MCP ツールを使ってステータスを設定します。これが推奨される方法で、人間と Claude の会話が線形に保たれるためです。

### Option B — UI で手動でステータスを変更する

タスクカードをタスク一覧の `reviewing` カラムから `done` カラムへドラッグします。ステータスは即座に更新されます。

どちらの方法でも、タスクのバッジは `done` になります。
