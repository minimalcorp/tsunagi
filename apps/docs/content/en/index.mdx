## The problem

Running Claude Code seriously means running it in **parallel**. Multiple
git worktrees, multiple terminals, multiple AI sessions — each working on a
different task at the same time. It's the same shape as managing a team of
human engineers as a PM.

But the moment you spawn five Claude sessions across five terminals, you
hit a wall:

- **You can't tell what's running where.** Which terminal owns which task?
  Who's coding what?
- **Claude's work is invisible.** You don't know what each session is
  thinking, what it's doing right now, or how far along it is.
- **Task management bleeds into your day.** Creating, updating, prioritizing
  tasks across multiple repos pulls you out of the work.

The benefit of parallelization is capped by your own cognitive load.

{/* ----------------------------------------------------------------- */}
{/* Solution */}
{/* ----------------------------------------------------------------- */}

## How Tsunagi solves it

Tsunagi treats every task as **its own git worktree, its own Claude
session, its own tab in one Web UI**. Claude's hooks stream the work into
that UI in real time, so you can see what every session is doing at a
glance.

You operate from the seat of a PM, not a stack of terminals.

<img
  src="/screenshots/task-detail.png"
  alt="Task detail with multiple Claude sessions"
  style={{ maxWidth: '100%', height: 'auto' }}
/>

And because Tsunagi exposes its own MCP server, **Claude itself can manage
your tasks** — create, update, reprioritize, sweep stale ones — so even the
PM work itself can be delegated to AI.

## Quick start

```bash
npx @minimalcorp/tsunagi
```

That's it. Tsunagi will boot the local web UI on
[http://localhost:2791](http://localhost:2791) and walk you through the
initial setup.

See [Installation](/en/getting-started/installation/) for prerequisites and
the [First Development Cycle tutorial](/en/tutorial/create-task/) for a
walkthrough of a complete task lifecycle.
