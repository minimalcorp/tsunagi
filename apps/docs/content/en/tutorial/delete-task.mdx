# 6. Delete the Task

Now that the task is `done` and verified, the last step is to delete it.
Deleting a task in Tsunagi removes:

- The task record from the database
- The git worktree from disk (`~/.tsunagi/workspaces/.../feat-hello-world/`)
- Any associated tab sessions

The bare repository is **not** affected — only the worktree directory.

## Ask Claude to delete it via MCP

In the still-open Claude tab, type:

```text
Please delete this task using the MCP tool.
```

Claude will call `tsunagi_delete_task` with the current task's ID. You'll
see:

1. The task disappear from the task list immediately
2. The worktree directory removed from the file system
3. The Claude tab closes (or shows a "task no longer exists" message)

This is one of the demonstrations of Tsunagi's MCP server: **Claude can
manage Tsunagi's own state on your behalf**.

<Callout type="info">
  You can also delete a task manually from the UI's task list (right-click or the task detail menu),
  but the MCP path is what makes Tsunagi's "delegate PM work to Claude" story real.
</Callout>

## Confirm cleanup

In another terminal:

```bash
ls ~/.tsunagi/workspaces/<owner>/<repo>/
```

The `feat-hello-world` directory should be gone. Only `.bare/` and
`.default/` remain.

## You did it

You just ran a complete Tsunagi development cycle:

1. Created a task → Tsunagi created the worktree
2. Started a Claude session → Claude planned and implemented
3. Watched real-time progress in the UI
4. Status moved automatically through `coding` → `reviewing`
5. Verified the result yourself
6. Marked the task done and asked Claude to delete it
