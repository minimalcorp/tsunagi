# MCP Server

Tsunagi exposes its task management as an
[MCP server](https://docs.claude.com/en/docs/claude-code/mcp). That means
**Claude itself can call into Tsunagi** to create, update, reorder, and
delete tasks.

This is the heart of Tsunagi's "delegate PM work to AI" story.

## Bundled tools

The Tsunagi plugin registers the following MCP tools (subject to change):

| Tool                              | Purpose                                        |
| --------------------------------- | ---------------------------------------------- |
| `tsunagi_create_task`             | Create a new task                              |
| `tsunagi_update_task`             | Update title, description, status, order, etc. |
| `tsunagi_delete_task`             | Delete a task and its worktree                 |
| `tsunagi_get_task`                | Fetch a task by id, session_id, or cwd         |
| `tsunagi_list_tasks`              | List tasks (filtered by status / repo / etc.)  |
| `tsunagi_list_repos`              | List registered repositories                   |
| `tsunagi_ensure_default_worktree` | Ensure the `.default` worktree is in sync      |

## Bundled skill and curator agent

Beyond the raw tools, the plugin ships a **skill** that teaches Claude
how to use them correctly (status transitions, branch naming, when to
update descriptions) and a **curator agent** for triage workflows like
"reorder backlog by priority" or "sweep stale tasks."

## How it gets installed

When Tsunagi starts, it runs:

```bash
claude plugin marketplace add <bundled marketplace dir>
claude plugin install tsunagi-plugin@tsunagi-marketplace --scope user
```

When Tsunagi exits, it runs the inverse uninstall. This ensures the plugin
state stays in sync with the running Tsunagi process. Tsunagi never edits
`~/.claude/settings.json` directly — only the official `claude` CLI is
used.
