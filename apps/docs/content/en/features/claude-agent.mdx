# Claude Agent & Tab Sessions

Each task in Tsunagi can host **multiple Claude Code sessions**, called
**tabs**. A tab is a PTY-backed terminal session that runs Claude inside
the task's worktree.

  src="/screenshots/worktree-tabs.png"
  alt="Multiple tabs in a task"
  style={{ maxWidth: '100%', height: 'auto' }}
/>

## Why multiple tabs

- Run one tab implementing a feature while another debugs the test suite
- Spin up a throwaway exploratory session without losing the main thread
- Compare two approaches side by side

All tabs share the same worktree, so file edits from one are immediately
visible to the others.

## How a tab is launched

When you click **+ New Tab**, Tsunagi:

1. Spawns a shell process via `node-pty` inside the task's worktree
2. Sets the environment variables defined in the Tsunagi settings page
3. Starts Claude Code in that shell, with the same credentials Claude
   Code itself uses
4. Wires Claude's hooks to the Tsunagi backend so the UI sees what's
   happening

## Resilience

Tabs are persisted across browser refreshes and brief network drops. If
you close the browser, the tab keeps running on the server side; reopen
Tsunagi and reconnect.
